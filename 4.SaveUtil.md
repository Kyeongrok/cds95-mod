# SaveUtil
세이브 데이터 관리 유틸리티 + α

## 개요
세이브 데이터와 관련된 몇 가지 기능을 제공합니다.
- 세이브 데이터 백업 기능을 제공합니다
- 최근 10개의 백업으로부터 복원 기능을 제공합니다
- 중단 데이터에서 재개했을 때 중단 데이터가 삭제되지 않도록 합니다
- TradeUtil 플러그인의 확장 세이브 데이터에도 대응합니다

## 대응 버전
CDS95.exe 버전에 대한 의존성은 없습니다.
모든 버전에서 동작할 것으로 예상되지만, Ver.1.1.0.0과 Ver.1.2.7.0 이외는 미확인입니다.

## 기능 상세

### 1) 세이브 데이터 백업 기능
메인 메뉴에 추가되는 「세이브 관리」 또는 단축키로 접근합니다.

「세이브 관리」의 「백업(退避)」을 선택하면, 날짜/시간을 파일명으로 하여 세이브 데이터를 ZIP 압축하고 CDS95Util 아래의 SaveDataBackUp 폴더에 저장합니다.
SaveDataBackUp 폴더의 내용이 10개를 초과하면 오래된 것부터 SaveDataBackUpOld로 이동됩니다.
SaveDataBackUpOld에는 백업이 무제한으로 쌓이므로, 필요 없어진 경우 수동으로 삭제해야 합니다.

CTRL + ALT + S를 누르면 「백업(退避)」과 동일한 처리를 합니다. 자주 사용할 기능이라 판단하여 단축키를 배정했습니다. 성공 시 효과음이 재생됩니다.

「세이브 관리」의 「덮어쓰기 백업(上書き退避)」을 선택하면 Temporary.zip이라는 이름으로 백업됩니다.
「백업(退避)」과 달리 실행할 때마다 같은 파일을 덮어씁니다. 이력을 남기고 싶지 않을 때 사용합니다.

### 2) 최근 10개 백업 복원 기능
「세이브 관리」의 「복귀」 하위 메뉴로, SaveDataBackUp 폴더에 있는 백업 목록을 표시합니다.
선택하면 해당 시점의 세이브 상태로 되돌립니다.
SaveDataBackUpOld로 이동된 백업에서 복원하려면 수동으로 해야 합니다. 일반 ZIP 파일이므로 OS 기능이나 압축 프로그램으로 해제할 수 있습니다.

확장자 .TMP 세이브 데이터가 없는 백업에서 복원할 경우, 설치 폴더 내의 .TMP 세이브 데이터를 수동으로 삭제해야 하므로 주의하세요.

### 3) 중단 데이터 유지 기능
중단 데이터는 일반적으로 일반 세이브를 할 때 삭제됩니다.
그 외 타이밍에서 삭제하려면 메뉴의 「세이브 관리」 → 「중단 삭제」를 선택합니다.

## 주의 사항
누적 캐릭터를 사용한 경우, 백업되어야 할 파일이 누락될 수 있습니다.
해당 기능은 직접 테스트하지 않았기 때문에 미확인입니다. 만약 백업되지 않는 파일을 발견하면 파일명을 알려달라고 요청하고 있습니다.

현재 백업 대상 파일은 다음 15개입니다:
- ACCDATA.CDS : 현재 캐릭터 누적 데이터
- SAVEDATA.CDS : 일반 세이브 데이터
- SAVEDATA.TMP : 중단 세이브 데이터
- ACCDATA0.ACC : 누적 캐릭터 1 데이터
- ACCDATA0.IDX : 누적 캐릭터 1 데이터
- ACCDATA1.ACC : 누적 캐릭터 2 데이터
- ACCDATA1.IDX : 누적 캐릭터 2 데이터
- ACCDATA2.ACC : 누적 캐릭터 3 데이터
- ACCDATA2.IDX : 누적 캐릭터 3 데이터
- ACCDATA3.ACC : 누적 캐릭터 4 데이터
- ACCDATA3.IDX : 누적 캐릭터 4 데이터
- ACCDATA4.ACC : 누적 캐릭터 5 데이터
- ACCDATA4.IDX : 누적 캐릭터 5 데이터
- EXTRADATA.CDS : TradeUtil 확장 세이브 데이터 (일반 세이브용)
- EXTRADATA.TMP : TradeUtil 확장 세이브 데이터 (중단 세이브용)

## 주의
PatchUtil 플러그인 기능을 사용하므로 PatchUtil.plugin이 필요합니다.
