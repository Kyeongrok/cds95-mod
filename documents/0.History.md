# 업데이트 내역

## 2019년 1월 25일
- TradeUtil에서 교역품 취급량을 변경할 수 있는 기능을 추가했습니다.
- 수량 제한이 없는 창고 기능을 제공하는 CollectionUtil을 추가했습니다.
- 확장 저장 데이터 형식을 변경했습니다. 이전 버전의 TradeUtil 데이터는 읽어 들이는 과정에서 자동으로 변환되어 그대로 이어서 사용할 수 있습니다.

## 2019년 1월 21일
- CPUPatch, HotelUtil, TradeUtil 플러그인을 Ver.1.4.0.0에 대응하도록 업데이트했습니다.

## 2019년 1월 20일
- DDrawWrapper 플러그인 사용 시 Windows Vista 등 일부 환경에서 프로그램이 비활성화된 상태로 일정 시간이 지나면 CDS95가 멈추는 현상을 수정했습니다. 원인은 아직 알 수 없지만, 멈추지 않게 하는 방법은 확인했기 때문에 임시적인 해결책을 적용했습니다. 직접 확인한 환경은 Vista뿐이었지만, Windows 10에서도 같은 현상이 발생한다는 보고가 있었습니다.
- TradeUtil의 「항구 시세」 기능명을 「도시 목록」으로 변경했습니다. 이와 함께 다음 기능을 추가했습니다.
  - 방문 완료 표시
  - 발견 완료 표시
  - 건설 완료 표시

  또한 항구가 아닌 도시도 표시할지 여부를 전환할 수 있도록 했습니다.
- 1월 19일 수정의 영향으로 패치 설정 화면을 닫을 때 체크 상태 변경이 인식되지 않던 문제를 수정했습니다.

## 2019년 1월 19일
- DDrawWrapper 플러그인에 다음과 같은 개선을 적용했습니다.
  - 프로그램이 비활성 상태일 때 Windows XP 이전 환경에서 다른 창의 잔상이 남지 않도록 수정했습니다.
  - 선박 구입이나 발견물 이벤트 영상 재생 시 화면 왼쪽 위 등에 영상의 첫 프레임이 깨진 채 남는 CDS95의 기존 버그를 완화하기 위해, 영상 재생 중에는 메인 화면을 암전시키도록 했습니다.
  - Windows 95 호환 모드 및 256색 모드를 감지하면 오류를 표시하고 실행되지 않도록 했습니다.
- CPUPatch 플러그인은 Windows Vista 이후 환경에서 대기 시간을 0.5ms로 줄이도록 변경했습니다.
- PatchUtil의 패치 설정 화면에서 다시 읽기를 한 뒤 체크 상태를 변경하지 않고 창을 닫으면 패치가 적용되지 않는 문제를 수정했습니다.
- 전체 소스 코드를 전반적으로 정리하고 수정했습니다. 겉으로 보이는 동작에는 특별한 변화가 없을 것입니다.

## 2019년 1월 16일
- DirectDraw를 에뮬레이션하는 DDrawWrapper 플러그인을 추가했습니다. 이제 256색 모드를 사용하지 않아도 게임을 실행할 수 있습니다. 이 플러그인이 활성화되어 있으면 DirectDraw를 전혀 사용하지 않습니다.
- CDS95.exe의 메모리 버그를 우회하는 MemoryFix 플러그인을 추가했습니다. 이를 통해 계산기 화면 등을 표시할 때 발생하던 CTD(Crash to Desktop) 현상이 사라집니다. 그 외의 상황에서도 CTD를 방지할 수 있을 것으로 기대됩니다. 이 기능 덕분에 Windows 95 호환 모드를 사용하지 않아도 실행할 수 있을 것으로 보입니다. 또한 DDrawWrapper와 함께 사용하면 Windows 10에서도 호환성 탭의 모든 옵션을 끈 상태로 안정적으로 실행할 수 있습니다.
- 윈도우 메시지 후킹 처리를 PatchUtil을 통해 수행하도록 통합했습니다. 이로 인해 PatchUtil에 의존하는 플러그인이 늘어났습니다.
- MouseUtil 사용 시 PatchUtil이나 TradeUtil의 화면에서 스크롤 막대를 드래그할 수 없던 문제를 수정했습니다.
- ddraw.dll에 남아 있던 디버그 코드 때문에 일부 시스템에서 실행되지 않던 문제를 수정했습니다.
- TradeUtil에서 새 게임(New Game) 시작 시 확장 저장 데이터가 그대로 이어지던 문제를 수정했습니다.

## 2019년 1월 15일
- Ver.1.4.0.0에서 게임 메인 창을 정상적으로 찾지 못하던 문제를 수정했습니다.
- 5ch 「대항해시대 Part36」 스레드의 869님, 버그 제보와 테스트 자료 제공에 감사드립니다.

## 2019년 1월 14일
- SaveUtil의 저장 관리 기능에서 하위 메뉴에서 정상적으로 복귀하지 못하던 문제를 수정했습니다.
- SaveUtil에서 누적 캐릭터 데이터를 올바르게 백업할 수 있도록 수정했습니다.
- HotelUtil의 숙박 일수 계산기에서 두 자리 이상의 숫자를 버튼으로 입력했을 때 동작이 이상하던 문제를 수정했습니다.
- TradeUtil에서 확장 저장 데이터가 없는 경우 투자액이 초기화되지 않던 문제를 수정했습니다.
- 5ch 「대항해시대 Part36」 스레드의 866님, 버그 제보 감사합니다.
