# CPUPatch

CPU 부하 감소 플러그인

## 개요

CPU 부하 감소를 수행하는 플러그인입니다.
CDS95.exe의 메시지 루프에 있는 busy loop(바쁜 대기) 부분에 적절히 대기(wait)를 넣습니다.

5ch의 「대항해시대 Part36」 스레드에서 835님이 작성하신 패치 정보 파일에 있는
「CPU 사용률 저감」을 대신할 목적으로 만든 것입니다.

## 대응 버전

현재로서는 Ver.1.2.7.0과 Ver.1.4.0.0 한정입니다.
아마도 835님이 만드신 「CPU 사용률 저감」이 원본 데이터가 FF 15로 시작하는 패치 주소만으로
동작하는 것으로 보이지만, 동작 확인이 되지 않았으니 양해 부탁드립니다.
본 플러그인 안에서 12 3D 0B 00으로 검색하여 나오는 부분을,
대상 버전에 맞는 위 835님의 주소로 바꿔 쓰면 동작할 것으로 생각되지만,
시도해 보실 경우에는 자기 책임 하에 진행해 주시기 바랍니다.
제대로 동작하지 않을 경우 프리즈나 크래시가 발생하니 주의하시기 바랍니다.

## 내부 동작

835님의 「CPU 사용률 저감」에서는 PeekMessage API로 메시지를 찾지 못했을 경우
Sleep API를 사용해 강제로 1ms 동안 대기를 추가하는 방식이지만,
본 플러그인에서는 busy loop 부분의 PeekMessage API 호출을
MsgWaitForMultipleObjectsEx API + PeekMessage API 호출로 대체했습니다.
최대 대기 시간은 1ms로 동일하지만, 메시지가 도착하면 즉시 대기가 종료되므로
사용자 조작에 대한 반응 속도가 더 개선되었을 것입니다.
또한 다른 부분의 처리에 악영향을 줄 만한 변경은 추가하지 않았습니다.
다만 어째서인지 가상 머신의 XP에서 동작시키면 반응이 둔한 느낌이 듭니다.
신경 쓰이시는 분은 본 플러그인을 제외해 주십시오.
아마도 MouseUtil 플러그인 쪽의 개선으로 해결할 수 있지 않을까 생각하고 있으므로,
언젠가 마음이 내키면 대응하겠습니다.

Windows Vista 이후에서는 NtSetTimerResolution과 SetWaitableTimer를 함께 사용하여
강제 대기 시간을 1회당 0.5ms로 줄일 수 있도록 했습니다.
이론적으로는 게임의 반응 속도가 더 빨라져야 하지만, 0.5ms 정도는 체감으로 알기 어렵겠지요.

## 주의

PatchUtil 플러그인의 기능을 사용하므로 PatchUtil.plugin이 있어야 합니다.

## 감사의 말

본 플러그인을 제작하면서 5ch의 「대항해시대 Part36」 스레드에서
835님이 작성하신 패치 정보 파일에 있는 「CPU 사용률 저감」을 힌트로 삼았습니다.
귀중한 분석과 패치 정보에 감사드립니다.
