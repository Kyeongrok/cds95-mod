# DDrawWrapper
DirectDraw 에뮬레이션 플러그인

## 개요
DirectDraw를 에뮬레이션하는 플러그인입니다.
이 플러그인을 사용하면 DirectDraw를 사용하지 않고, 화면 출력을 GDI(Graphics Device Interface) 방식으로 대체합니다.
또한 색상 변환(Color Conversion)을 지원하며, 디스플레이 모드 검사 우회 기능도 구현되어 있으므로, 256색 모드에서만 실행되는 게임도 풀컬러(High Color/True Color) 모드에서 그대로 실행할 수 있습니다.

## 지원 버전
CDS95.exe의 버전에 의존하지 않습니다.
원칙적으로 모든 버전에서 동작하도록 설계되었지만, 현재 확인된 버전은 다음과 같습니다.
- Ver.1.1.0.0
- Ver.1.2.3.0
- Ver.1.2.7.0
- Ver.1.4.0.0

그 외 버전은 아직 동작 여부가 확인되지 않았습니다.

## CDS95에서의 동작
DirectDraw 때문에 발생하던 다음과 같은 문제를 개선합니다.
- 게임 창 바깥을 클릭했을 때 발생하는 프리징 현상이 사라집니다.

Ver.1.1.0.0에서는 별도의 패치를 적용하지 않아도 해당 문제가 해결되는 것을 확인했습니다.

## 기타
이 플러그인은 특정 게임에 의존하지 않는 범용적인 기법을 사용하고 있으므로, 다른 게임에도 응용할 가능성이 있습니다.
다만 현재는 CDS95에서 사용하는 DirectDraw 기능만 구현되어 있기 때문에, 현 상태에서는 다른 게임에 그대로 적용할 수는 없습니다.
앞으로 지원 게임을 추가할 수도 있습니다. 『대항해시대 II』도 256색 게임이었던 것으로 기억하므로, 향후 해당 게임도 지원할 가능성이 있습니다.

## 업데이트 내역

### 2019/01/19
게임 창이 비활성 상태일 때 Windows XP 이전 환경에서 다른 창의 잔상이 남지 않도록 수정했습니다.
선박 구매나 발견물(이벤트)의 동영상 재생 시, 화면 왼쪽 위 등에 동영상의 첫 프레임이 깨진 상태로 남는 CDS95 고유의 버그가 보기 좋지 않아, 동영상 재생 중에는 메인 화면을 어둡게 처리하도록 변경했습니다.
Windows 95 호환 모드 또는 256색 모드에서 실행되는 경우 이를 감지하여 오류 메시지를 표시하고 실행되지 않도록 변경했습니다.

### 2019/01/20
Windows Vista 등 일부 환경에서 게임 창이 비활성 상태로 일정 시간이 지나면 CDS95가 멈추는 현상에 대응했습니다.
원인은 아직 밝혀내지 못했지만, 프리징이 발생하지 않도록 하는 방법은 확인했기 때문에 우선 임시 대응 방식으로 해결했습니다.
개발자가 직접 확인한 환경은 Windows Vista뿐이지만, Windows 10에서도 비슷한 문제가 발생했다는 보고가 있었습니다.

## 이 문서에서 얻을 수 있는 중요한 정보
이 문서는 한글판 적용과 관련해서도 몇 가지 중요한 사실을 알려줍니다.

DDrawWrapper는 CDS95의 메모리 구조를 수정하는 플러그인이 아니라 DirectDraw API를 대체하는 래퍼(wrapper)입니다.
따라서 일본판과 한글판의 EXE 차이에 거의 영향을 받지 않는 플러그인입니다.
실제로 제작자도 "CDS95.exe의 버전에 의존하지 않는다"고 명시하고 있으므로, 한글판에서도 비교적 그대로 사용할 가능성이 높습니다.

반면 TradeUtil, HotelUtil, CollectionUtil처럼 게임 내부 주소를 참조하는 플러그인과는 성격이 완전히 다릅니다.

지금까지 번역한 README들을 종합하면 플러그인은 크게 두 종류로 나눌 수 있습니다.

**버전 독립형 플러그인**
- DDrawWrapper
- MemoryFix
- (일부) CPUPatch

**게임 내부 메모리 주소에 의존하는 플러그인**
- TradeUtil
- HotelUtil
- CollectionUtil
- SaveUtil
- MouseUtil

즉, 한글판에 포팅해야 하는 대상은 대부분 두 번째 부류이며, DDrawWrapper는 특별한 수정 없이도 사용할 수 있을 가능성이 가장 높은 플러그인으로 보입니다.
