# PatchUtil
범용 동적 패치 플러그인

## 개요
실행 중인 코드에 동적으로 패치를 적용하는 범용 플러그인입니다.
EXE 파일 자체를 미리 수정하지 않고도 메모리 상에서 직접 패치를 적용할 수 있습니다.
또한 UI를 통해 패치를 켜거나 끌 수 있으며, 설정 파일을 다시 불러와 실행 중에도 즉시 패치를 변경할 수 있습니다.
콘솔 게임 에뮬레이터의 ROM 패치(치트 코드)와 비슷한 개념이라고 생각하면 됩니다.

단, 메모리를 일정 주기로 계속 덮어쓰는 기능(메모리 고정)은 제공하지 않습니다. 그러한 용도에는 다른 프로그램을 사용하시기 바랍니다.
또한 이 플러그인은 다른 플러그인들이 일부 기능을 호출하여 사용하므로 절대로 삭제하지 마십시오. 설정 파일을 만들지 않는다면 존재만으로는 아무런 부작용이 없습니다.

## 지원 버전
플러그인 자체는 CDS95.exe의 버전에 의존하지 않습니다.
원칙적으로 모든 버전에서 동작하도록 만들어졌지만, 현재 확인된 버전은 Ver.1.1.0.0, Ver.1.2.7.0뿐입니다.
단, 패치 정보 파일의 이름과 내용은 사용하는 CDS95.exe 버전에 맞아야 합니다.

## 사용 방법
`CDS95Patch####.txt` (####는 4자리 버전 번호) 형식의 패치 파일을 만들어 CDS95Util 폴더에 넣으면 프로그램이 자동으로 읽어서 패치를 적용합니다.
예를 들어 Ver.1.2.7.0이라면 `CDS95Patch1270.txt`가 됩니다.

5ch 「대항해시대 Part36」의 835님이 작성한 패치 파일도 거의 그대로 사용할 수 있습니다.
예를 들어 `대항해시대3개조1270.txt`를 `CDS95Patch1270.txt`로 이름만 변경하면 대부분의 패치를 그대로 사용할 수 있습니다.

새로운 패치 파일이 발견되면 게임 시작 시 패치 선택 화면(UI)이 표시됩니다. 사용할 패치에 체크하면 됩니다.
체크 상태는 설정 파일에 저장되므로 다음부터는 같은 화면이 나타나지 않습니다. 새로운 패치를 추가하면 다음 실행 시 다시 선택 화면이 나타납니다.

CDS95 메인 메뉴에는 「패치(Patch)」 메뉴가 추가됩니다. 여기서 언제든지 패치 선택, 패치 파일 다시 읽기를 할 수 있습니다.

패치를 켜거나 끄는 작업은 패치의 종류에 따라 게임 실행 중에 변경하면 충돌(Crash)이나 프리징(Freeze)이 발생할 수 있습니다. 따라서 반드시 세이브한 후 변경하시기 바랍니다.

## 사용할 수 없는 패치
Import Table(IAT)를 수정하는 패치는 사용할 수 없습니다. 예를 들면 「CPU 사용률 감소」, 「CPU 사용률 감소 Ver.2」 패치가 이에 해당합니다.
이러한 패치는 자동으로 감지되어 UI에서 빨간색 배경으로 표시됩니다.
CPU 사용률 감소 기능은 대신 CPUPatch 플러그인으로 제공됩니다.

감지하지 못하는 경우도 있기 때문에 사용 불가능한 패치가 더 있을 수도 있습니다. EXE를 직접 수정하면 정상인데 PatchUtil에서는 충돌이나 프리징이 발생하는 패치를 발견하면 제보를 부탁드립니다.

또 하나의 사용 불가 조건은 이미 EXE의 해당 주소가 수정되어 있는 경우입니다.
PatchUtil은 패치를 적용하기 전에 원본 데이터가 패치 파일에 기록된 값과 같은지를 확인합니다.
따라서 이미 EXE를 수정한 경우나 다른 패치가 먼저 적용된 경우 역시 빨간색 배경으로 표시됩니다.

## 패치 파일 형식
다음 기호로 시작하는 줄은 패치 제목으로 인식됩니다: `◆ ◇ ■ □ ● 〇`
제목부터 다음 제목까지의 내용은 해당 패치의 데이터로 처리됩니다.
제목도 아니고 16진수도 아닌 줄은 무시됩니다.
16진수(0~9, A~F)로 시작하는 줄은 패치 데이터로 인식됩니다.

**패치 데이터 형식**: `주소 : 원래 데이터 → 변경 데이터`

다음 순서로 해석합니다.
- 줄 처음부터 ":"까지를 주소로 인식
- 뒤의 16진수를 원본 데이터로 인식 (공백은 무시)
- 16진수가 아닌 문자가 나오면 원본 데이터 종료
- 다음 16진수부터 변경 데이터 시작, 다시 16진수가 아닌 문자가 나오면 종료
- 그 이후 내용은 무시
- 원본과 변경 데이터 길이가 다르면 오류
- 0x가 있든 없든 모두 16진수로 처리

**주소 표현**

주소는 다음 네 가지 형식을 사용할 수 있습니다.
- BaseAddress
- BaseAddress + Offset
- BaseAddress + RecordSize * Index
- BaseAddress + RecordSize * Index + Offset

이 네 가지만 지원합니다.

예를 들어 캐랙(Carrack)의 가격을 5배로 변경하는 경우:
```
0xC6600 + 0x40 * 0x02 + 0x38:
28 00 00 00 → 96 00 00 00
```
의 의미는 다음과 같습니다.
- 0xC6600 = 선박 데이터 시작 주소
- 0x40 = 선박 레코드 크기
- 0x02 = 캐랙의 인덱스
- 0x38 = 가격 필드

## 835님의 패치 파일을 사용하는 경우
다음 패치는 사용할 수 없으므로 삭제해야 합니다.
- CPU 사용률 감소
- CPU 사용률 감소 Ver.2

다음 패치(※ 대항해시대2의 인물을 고용 가능하게 하는 패치)는 들여쓰기가 되어 있어 인식되지 않습니다.
들여쓰기를 제거한 뒤 ※를 ◆ 등으로 변경하면 정상 인식됩니다.

1.2.7.0 이외의 버전은 확인되지 않았기 때문에 동작하지 않는 패치가 있을 수도 있습니다.

세이브 데이터를 수정하는 패치는 PatchUtil에서는 지원하지 않습니다.
게임 도중 적용하기 위해 세이브 파일 수정이 필요한 패치는 바이너리 에디터 등을 이용해 직접 수정해야 합니다.

## 주소 변환 기능
메인 메뉴의 「패치」 화면에는 「주소 변환(Address Convert)」 버튼이 있습니다.
이 기능은 EXE 파일의 오프셋과 실행 중 메모리 주소를 서로 변환해 주는 기능입니다.
메모리 주소는 「EXE 시작 주소 + 파일 오프셋」처럼 단순 계산으로 구할 수 없기 때문에 개발 보조 기능으로 제공됩니다.
범위를 벗어나면 00000000이 표시됩니다.

현재는 CDS95.exe가 사용하는 메모리 영역만 변환 가능합니다. 다른 DLL까지 확장할 수도 있지만 실질적으로 사용할 일은 거의 없을 것입니다.

## 감사의 말씀
본 플러그인을 제작하는 과정에서 5ch 「대항해시대 Part36」의 835님이 작성한 패치 파일을 테스트 데이터로 사용하였습니다.
귀중한 분석 자료와 패치 정보를 제공해 주셔서 감사합니다.
